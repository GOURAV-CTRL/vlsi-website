Multiplexer — The Basics

A MUX selects one of several inputs and routes it to a single output, based on a select signal — think of it as a data selector, or a many-to-one switch.

An n-bit select line can choose among 2ⁿ inputs — 1 select bit → 2:1 MUX, 2 select bits → 4:1 MUX, and so on.

selOutput =
00in0
01in1
10in2
11in3
4:1 MUX — Behavioral, Using case
module mux4to1 ( input wire [7:0] in0, in1, in2, in3, input wire [1:0] sel, output reg [7:0] y ); always @(*) begin case (sel) 2'b00: y = in0; 2'b01: y = in1; 2'b10: y = in2; 2'b11: y = in3; default: y = 8'bx; // unreachable, but keeps tools happy endcase end endmodule
🔹 Key Points
  • Since sel is 2 bits, all four combinations are already covered — the default is technically unreachable but still good practice.
  • This style scales cleanly to wider MUXes just by adding more case branches — no nested if-else needed.
2:1 MUX — Dataflow, Using the Ternary Operator

For a simple 2:1 MUX, a one-line assign with the ternary (?:) operator is often cleaner than a full always/case block.

module mux2to1 ( input wire sel, input wire [7:0] in0, in1, output wire [7:0] y ); assign y = sel ? in1 : in0; // sel=1 -> in1, sel=0 -> in0 endmodule

★ Ternary operators can be nested to build wider MUXes (sel[1] ? (sel[0] ? in3 : in2) : (sel[0] ? in1 : in0)), but this gets unreadable fast — case statements are usually clearer beyond 2:1.

Parameterized N:1 MUX — Using a for Loop

Combining Part 8's for loop with an array of inputs lets you build a MUX of any width without hardcoding each branch.

module mux_n #( parameter WIDTH = 8, parameter N = 4, parameter SELW = 2 )( input wire [WIDTH-1:0] in [0:N-1], input wire [SELW-1:0] sel, output reg [WIDTH-1:0] y ); integer k; always @(*) begin y = {WIDTH{1'bx}}; for (k = 0; k < N; k = k + 1) begin if (sel == k) y = in[k]; end end endmodule

★ in here is an unpacked array of N vectors — each in[k] is a full WIDTH-bit bus, letting one module definition serve as a 2:1, 4:1, or 8:1 MUX just by changing parameters.

Demultiplexer — The Basics

A DEMUX is the mirror image of a MUX — it takes one input and routes it to exactly one of several outputs, chosen by the select line; every unselected output is driven to 0 (or held inactive).

selActive Output
00out0 = in, others = 0
01out1 = in, others = 0
10out2 = in, others = 0
11out3 = in, others = 0
1:4 DEMUX — Behavioral, Using case
module demux1to4 ( input wire in, input wire [1:0] sel, output reg out0, out1, out2, out3 ); always @(*) begin // default: everything OFF, avoids inferred latches out0 = 1'b0; out1 = 1'b0; out2 = 1'b0; out3 = 1'b0; case (sel) 2'b00: out0 = in; 2'b01: out1 = in; 2'b10: out2 = in; 2'b11: out3 = in; endcase end endmodule
🔹 Key Points
  • All four outputs are pre-assigned to 0 before the case statement — this guarantees every output has a defined value for every sel combination, so no default branch is even needed inside the case itself.
  • Only the selected output gets overwritten with in; every other output keeps its pre-assigned 0.
1:4 DEMUX — Bus-Style Output (Common Alternative)

Instead of four separate output ports, many designs bundle the outputs into a single bus — this is especially convenient with a shift-based one-hot trick.

module demux1to4_bus ( input wire in, input wire [1:0] sel, output reg [3:0] out ); always @(*) begin out = 4'b0000; out[sel] = in; // only the selected bit gets driven end endmodule

★ out[sel] is a great example of a variable bit-select — Verilog allows the index itself to be a signal, not just a constant, exactly like the shift-amount trick from the very first notes on shift operators.

Important Warnings
⚠️ DEMUX Needs Every Output Pre-Assigned

Because a DEMUX has multiple outputs but only one is written per branch, forgetting to pre-assign all outputs to 0 before the case statement causes every unselected output to infer a latch — this is easy to miss compared to a MUX, which only has one output to worry about.

⚠️ Unpacked Arrays as Ports Need Tool Support

The parameterized mux_n style using input wire [WIDTH-1:0] in [0:N-1] (an array port) is valid SystemVerilog/Verilog-2001 but not all tools/flows accept array ports cleanly — check your simulator/synthesizer before relying on it in a real IP.

⚠️ Nested Ternaries Get Unreadable Fast

While a nested ternary can technically build any width of MUX in one assign statement, beyond 2:1 it becomes hard to verify by eye — prefer a case statement in an always block for anything wider.

⚠️ out[sel] Needs sel to Be In-Range

Using a variable index like out[sel] relies on sel always being a valid index for the output width — an out-of-range sel value (possible if sel is wider than needed) simply has no effect rather than throwing an error, which can hide bugs in simulation.
