Gate Level Modeling — 4-bit Ripple Carry Adder & BCD Adder

1️⃣ Ripple Carry Adder (RCA) Design

→ Concept

  • An RCA is built using four full adders connected in series.
  • The carry-out of each stage becomes the carry-in of the next stage — hence the name "ripple" carry.
  • This is the simplest way to build a multi-bit adder, but it's slow for large bit-widths since the carry must "ripple" through every stage before the final sum is correct.

→ Implementation

  • Each full_adder module is instantiated individually (Gate Level style — built from basic logic gates like AND, OR, XOR).
  • Vectors are used to handle multi-bit inputs A[3:0], B[3:0] and outputs Sum[3:0], Carry.
  • Internal wires are declared to connect the carry-out of one full adder to the carry-in of the next.
Remember: in GLM, you connect gates directly — no behavioral always blocks. It's like wiring a real circuit on paper!

→ Verification

  • The RCA module is set as the top module in the simulator/synthesis tool.
  • The elaborated schematic is checked to confirm all gate-level connections (wires, ports) are correctly wired.
  • This is a structural check only — it confirms wiring, not logical correctness.

2️⃣ BCD (Binary Coded Decimal) Adder Design

→ Concept

  • A BCD adder takes two 4-bit binary adders + extra correction logic.
  • Since BCD digits only go from 0–9, any binary sum greater than 9 needs to be corrected back into valid BCD form.
  • Correction = adding 6 (0110) to the result whenever the sum exceeds 9 or a carry is generated.

→ Implementation Steps

  • Step 1: First RCA adds the two 4-bit binary inputs → produces an initial sum (S3 S2 S1 S0) and carry.
  • Step 2: Logic gates (AND, OR) examine output bits S3, S2, S1 to detect if the result is greater than 9.
    ↳ Condition typically checks for combinations like (S3 AND S2) OR (S3 AND S1) OR Carry-out.
  • Step 3: If the condition is true, a correction value of 6 is added using a second 4-bit adder.

→ Design Nuances

  • Some bits of the correction adder's input are fixed at 0, others come directly from the logic gates' outputs.
  • The assign keyword is used in Verilog to wire constants or gate outputs directly to specific bit positions (continuous assignment).
  • Example pattern: assign correction[3:0] = {1'b0, carry_detect, carry_detect, 1'b0};
The correction logic is the "brain" of the BCD adder — without it, the circuit is just a plain binary adder!

3️⃣ Key Takeaways

  • Visualization ≠ Functionality: A correct-looking schematic only proves the structure is wired properly — it does NOT prove the logic behaves correctly.
  • Test Benches are essential: To truly verify functional correctness, you must simulate the design using test benches that apply input vectors and check outputs.
  • Future sessions will focus on writing test benches for these adder designs.

📝 Extra Concepts Worth Remembering

  • Full Adder basics: Sum = A ⊕ B ⊕ Cin, Carry = (A·B) + (Cin·(A⊕B))
  • Gate Level Modeling (GLM): Lowest level of abstraction in Verilog — describes circuits using primitive gates (and, or, xor, not, nand, nor, xnor) instead of behavioral code.
  • Why BCD matters: Used in digital clocks, calculators, and display systems where decimal representation is required.
  • Ripple delay tradeoff: RCA is simple but has propagation delay ∝ number of bits. Carry Look-Ahead Adders solve this speed issue (good topic to explore next!).
  • Module reuse: Notice how the same full_adder / 4-bit adder module is instantiated multiple times — a key idea of modular, reusable hardware design.
