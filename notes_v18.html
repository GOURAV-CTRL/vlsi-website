SR Latch 🔒

All About VLSI · Lecture Notes — Part 11: Sequential Building Blocks
What Makes a Latch a Latch

A latch is the simplest possible memory element — a circuit that can hold (store) one bit of state even after its inputs go away, unlike pure combinational logic which only reflects its current inputs.

Unlike a flip-flop (covered in Part 12), a latch is level-sensitive, not edge-sensitive — it stays "transparent" and follows its inputs the entire time its enable is active, only "latching" the last value once enable goes low.

SR Latch — Behavior

The SR latch (Set-Reset latch) has two inputs, S (set) and R (reset), and one stored output q (plus its complement, qn).

SRq (next state)Meaning
00hold (no change)Latch keeps its previous value
010Reset — q forced low
101Set — q forced high
11invalidForbidden — both outputs forced same value
SR Latch — Behavioral Code

Because this circuit must remember its state, the always block deliberately leaves q unassigned in the "hold" case — this is the one place in RTL where an inferred latch is intentional, not a bug.

module sr_latch ( input wire s, r, output reg q, output wire qn ); always @(*) begin if (s && !r) q = 1'b1; // Set else if (!s && r) q = 1'b0; // Reset else if (!s && !r) q = q; // Hold — deliberately no change, latch inferred else q = 1'bx; // S=R=1: invalid/undefined state end assign qn = ~q; endmodule
🔹 Key Points
  • q = q; in the hold branch is what makes the synthesizer infer storage (a latch) instead of pure combinational logic — normally this exact pattern is a bug (Part 8's warning), but here it's the entire point.
  • The S=R=1 case is marked invalid because in real cross-coupled gate hardware, both q and qn would be forced to the same value, breaking the qn = ~q relationship.
  • qn is derived combinationally from q here for simplicity — a true gate-level SR latch generates qn directly from cross-coupled NOR/NAND gates instead.
Gated SR Latch — Adding an Enable

A gated (clocked) SR latch adds an enable input — while enable is high the latch is transparent and behaves like the plain SR latch above; while enable is low, q holds its value regardless of S and R.

module sr_latch_gated ( input wire s, r, en, output reg q ); always @(*) begin if (en) begin if (s && !r) q = 1'b1; else if (!s && r) q = 1'b0; else if (!s && !r) q = q; // hold within active window else q = 1'bx; end // en == 0: no assignment at all -> q holds (latch) end endmodule

★ Notice there's no else branch for en == 0 at all — leaving q completely unassigned in that case is exactly what makes the enable behave like a real gated latch, freezing q the instant en drops.

Gate-Level View — NOR vs NAND SR Latch

Cross-coupled NAND gatesActive-low S̄, R̄S̄ = R̄ = 1

The behavioral code above models the active-high NOR-based version. For the NAND-based version, simply invert the sense of S and R in the conditions (or rename them s_n, r_n) and swap the idle/forbidden rows accordingly.

Important Warnings
⚠️ S = R = 1 Is Genuinely Forbidden

Unlike a "don't care" case, the S=R=1 input combination is a real hardware hazard in a basic SR latch — both q and qn get forced toward the same logic level, and when the inputs return to S=R=0 together, the final state becomes unpredictable (a race) depending on gate delays.

⚠️ This Is One of the Few Places a Latch Is Intentional

Every earlier lecture (Parts 6–10) warned that a missing assignment or a q = q; self-hold pattern causes an unwanted inferred latch — here it's flipped: the SR latch's entire purpose is that inferred storage, so don't "fix" it by adding a default case that always assigns q.

⚠️ Latches Are Level-Sensitive, Not Edge-Sensitive

A gated SR latch stays transparent for the entire time en is high — any glitch on s or r during that window passes straight through to q — this is very different from a flip-flop, which only samples inputs at a clock edge (see Part 12).

⚠️ Latches Are Generally Avoided in Synchronous RTL

Because of their level-sensitive, glitch-prone nature, most digital design flows explicitly avoid inferring latches in synthesizable RTL except in very specific, deliberate cases like this one — most synthesis tools will flag inferred latches with a warning precisely so they don't slip in by accident.
