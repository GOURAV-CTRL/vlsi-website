BCD Adder 🔢

What Is BCD, and Why Adding It Is Tricky

BCD (Binary-Coded Decimal) represents each decimal digit (0–9) using its own 4-bit binary pattern — so a 2-digit decimal number needs 8 bits, one nibble per digit, never using the values 1010–1111.

A plain 4-bit binary adder happily produces sums from 0000 to 1111 — but any BCD digit sum above 1001 (9) is invalid and must be corrected back into proper decimal form.

The Correction Rule

First add the two 4-bit BCD digits (plus any carry-in) as an ordinary binary adder, giving a 5-bit result. Then check:

  • If the binary sum is ≤ 9 and there's no carry-out — it's already valid BCD, no correction needed.
  • If the binary sum is ≥ 10, or the binary addition produced a carry-out — the result is invalid BCD, and you must add 6 (0110) to skip over the unused codes 1010–1111.

Adding 6 either creates a new carry-out (the correct decimal carry) or pushes the result back into the valid 0–9 range — this is the entire trick behind a BCD adder.

Binary SumCarry from Stage 1CorrectionResult
0000 – 1001 (0–9)0NoneValid BCD digit
1010 – 1111 (10–15)0+ 0110Valid BCD digit, cout = 1
0000 – 1001 (16–25 total)1+ 0110Valid BCD digit, cout = 1
Method 1 — Pure Behavioral (Single Digit)

The most direct way to describe a single-digit BCD adder: do the binary add, check the correction condition, then add 6 if needed — all inside one always block.

module bcd_adder ( input wire [3:0] a, b, // two BCD digits (0-9 each) input wire cin, // carry-in from previous digit output reg [3:0] sum, // corrected BCD digit output reg cout // carry-out to next digit ); reg [4:0] bin_sum; // 5-bit binary sum, room for overflow always @(*) begin bin_sum = a + b + cin; // step 1: plain binary add if (bin_sum > 9) begin sum = bin_sum + 6; // step 2: correction cout = 1'b1; end else begin sum = bin_sum[3:0]; // already valid, just truncate cout = 1'b0; end end endmodule
🔹 Key Points
  • bin_sum is 5 bits wide so it can hold values up to 9+9+1 = 19 without overflow.
  • The condition bin_sum > 9 alone is enough — it automatically also covers the case where the plain binary add produced its own carry-out, since that pushes bin_sum ≥ 16.
  • sum only needs the lower 4 bits after correction, since a valid BCD digit always fits in 4 bits.
Method 2 — Built from Two 4-Bit Adders

This mirrors the classic textbook diagram: a 4-bit binary adder computes the raw sum, a small piece of logic decides whether correction is needed, and a second 4-bit adder adds 6 conditionally.

module bcd_adder_structural ( input wire [3:0] a, b, input wire cin, output wire [3:0] sum, output wire cout ); wire [3:0] bin_sum; wire c1; // carry out of first binary add wire correction_needed; wire [3:0] corrected_sum; wire c2; // carry out of correction add // Stage 1: plain 4-bit binary add assign {c1, bin_sum} = a + b + cin; // Decide correction: needed if c1=1, or bin_sum > 9 (i.e. bits 3&2 or 3&1 set) assign correction_needed = c1 | (bin_sum[3] & (bin_sum[2] | bin_sum[1])); // Stage 2: conditionally add 0110 assign {c2, corrected_sum} = bin_sum + (correction_needed ? 4'b0110 : 4'b0000); assign sum = corrected_sum; assign cout = correction_needed; // final decimal carry-out endmodule

★ The Boolean expression c1 | (bin_sum[3] & (bin_sum[2] | bin_sum[1])) is the standard gate-level shortcut for detecting "sum > 9" without doing a full magnitude comparison — it's derived directly from which bit patterns 10–15 share.

Chaining for Multi-Digit BCD Numbers

Just like a ripple-carry binary adder, a multi-digit BCD adder is built by chaining single-digit bcd_adder modules, feeding each stage's cout into the next stage's cin.

module bcd_adder_2digit ( input wire [7:0] a, b, // two 2-digit BCD numbers input wire cin, output wire [7:0] sum, output wire cout ); wire c_mid; bcd_adder digit0 (.a(a[3:0]), .b(b[3:0]), .cin(cin), .sum(sum[3:0]), .cout(c_mid)); bcd_adder digit1 (.a(a[7:4]), .b(b[7:4]), .cin(c_mid), .sum(sum[7:4]), .cout(cout)); endmodule

★ Each nibble of a and b must independently represent a valid digit 0–9 — feeding in raw non-BCD binary values through this chain gives meaningless results.

Important Warnings
⚠️ bin_sum Must Be Wide Enough

Declaring the intermediate sum as only 4 bits silently truncates values above 15, making the > 9 correction check see wrapped-around garbage instead of the real sum — always give it at least 5 bits.

⚠️ cout Is Not the Same as the Binary Adder's Carry

The final cout of a BCD digit means "carry into the next decimal digit," not "the raw binary adder overflowed" — these coincide in some cases but the correction step can also generate cout on its own, so always drive cout from the correction logic, not the Stage-1 adder directly.

⚠️ Invalid BCD Inputs Break Everything

If a or b is ever a code from 1010–1111 (not a valid decimal digit), the correction logic has no way to detect or fix that — validate/constrain inputs upstream, especially in a testbench driving random 4-bit values.

⚠️ This Is Still Combinational

Despite "adder" sounding sequential, a BCD adder (single or multi-digit, ripple style) is pure combinational logic — no clock is involved, and propagation delay simply ripples through each chained digit just like a binary ripple-carry adder.
