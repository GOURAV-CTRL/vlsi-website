2-Bit Comparator — Gate Level Modelling 📒

All About VLSI · Lecture Notes
What is a Magnitude Comparator?

A magnitude comparator compares two binary numbers, A and B, and tells us which one is bigger, smaller, or equal.

  • Has 3 outputs: A>B, A<B, A=B — exactly one is high at a time.
  • A 2-bit comparator takes two 2-bit inputs → A = A1 A0 and B = B1 B0.
  • Built purely from basic gates (XNOR, AND, OR, NOT) here — no always block, no behavioural code.
Boolean Expressions

First, define equality bits for each position using XNOR (since XNOR = 1 only when both bits match):

x0 = ~(A0 ^ B0); // A0 == B0 x1 = ~(A1 ^ B1); // A1 == B1
🔹 A = B (Equal)

Both bit positions must match → AND the two equality bits:

A_eq_B = x1 & x0;
🔹 A > B (Greater)

A is bigger if its MSB is bigger, OR the MSBs are equal and its LSB is bigger:

A_gt_B = (A1 & ~B1) | (x1 & A0 & ~B0);
🔹 A < B (Less)

By symmetry — swap the roles of A and B from the greater-than expression:

A_lt_B = (~A1 & B1) | (x1 & ~A0 & B0);

★ These three outputs are mutually exclusive — exactly one of them is 1 for any given A and B.

Verilog Code (Gate Level)
module comparator_2bit ( input A1, A0, B1, B0, output A_gt_B, A_lt_B, A_eq_B ); wire x0, x1; wire n_A1, n_A0, n_B1, n_B0; wire t1, t2, t3, t4; // equality of each bit pair xnor (x0, A0, B0); xnor (x1, A1, B1); // A = B and (A_eq_B, x1, x0); // inverted signals for the compare logic not (n_A1, A1); not (n_A0, A0); not (n_B1, B1); not (n_B0, B0); // A > B and (t1, A1, n_B1); and (t2, x1, A0, n_B0); or (A_gt_B, t1, t2); // A < B and (t3, n_A1, B1); and (t4, x1, n_A0, B0); or (A_lt_B, t3, t4); endmodule

★ Every line here is a primitive gate instantiation (and, or, not, xnor) — this is the whole point of gate level modelling: build the circuit exactly the way you'd wire it on a breadboard.

⚠️ Important Warnings
⚡ Don't Mix Modelling Styles

In pure gate level modelling, avoid using assign or always blocks — only primitive gates (and, or, not, xor, xnor, nand, nor) are allowed. Mixing styles defeats the purpose of the exercise.

⚡ Gate Instance Names are Optional

You can write and(y, a, b); without an instance name — Verilog auto-generates one. Named instances like and g1(y, a, b); just make debugging easier in waveform viewers.

⚡ Watch the Output Port Order

In gate primitives, the output always comes first in the port list, followed by the inputs — e.g. and(out, in1, in2), not the other way around. Easy to mix up when converting from equations.
