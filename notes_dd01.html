Number System Conversions

Digital Design — Module 01, Video 01

Digital systems understand only two voltage levels — low and high — written as 0 and 1. Everything a chip does is built on top of this two-symbol language, so before touching gates or circuits we need to be fluent in the number systems used to represent data.

1The Four Number Systems

There are four systems we work with constantly in digital design, each defined by its radix (the number of unique digits it uses):

SystemRadixDigits Used
Binary20, 1
Octal80 – 7
Decimal100 – 9
Hexadecimal160 – 9, A – F

Binary is the native language of digital machines because a transistor can only cleanly represent two states — on or off. Octal and hexadecimal exist mainly as shorthand, letting humans write long binary strings more compactly.

2Octal ⇄ Binary

Since 8 = 2³, every octal digit maps neatly onto exactly 3 bits of binary. This clean relationship is what makes octal conversions so mechanical.

Octal → Binary
Replace each octal digit with its 3-bit binary equivalent.
Example: 5 2 7 → 101 010 111
Binary → Octal
Starting from the binary point, group bits into sets of 3 (pad with zeros if needed).
Convert each group to its octal digit.
Watch out — For fractional numbers, grouping starts from the binary point and moves outward in both directions — group left of the point toward the MSB, and right of the point toward the LSB, padding zeros wherever a group is incomplete.

3Octal ⇄ Hexadecimal

There's no direct shortcut digit-to-digit relationship here, so binary is used as the middleman:

Octal → Hexadecimal
Octal → Binary (groups of 3) → regroup the same bit string into groups of 4 → Hexadecimal.
Hexadecimal → Octal
Hex → Binary (groups of 4) → regroup into sets of 3 → Octal.

The only thing that changes between the two directions is the grouping size — 3 bits for octal, 4 bits for hex — since 16 = 2⁴.

4Decimal ⇄ Binary

Decimal → Binary
Repeatedly divide the number by 2, recording each remainder.
Read the remainders bottom-to-top to get the binary result.
For fractional parts, repeatedly multiply by 2 instead, and read the integer part generated at each step top-to-bottom.
Binary → Decimal
Multiply each bit by its positional weight (powers of 2) and sum the results.
Example: 1011 = 1·2³ + 0·2² + 1·2¹ + 1·2⁰ = 8 + 0 + 2 + 1 = 11
Key takeaway — Every conversion in this video boils down to the same idea: match up group sizes to the radix relationship between two systems, and use binary as the universal translator when there's no direct shortcut.
