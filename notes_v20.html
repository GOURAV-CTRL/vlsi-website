SISO & SIPO Shift Registers ➡️

All About VLSI · Lecture Notes — Part 13: Shift Registers
What a Shift Register Actually Is

A shift register is just a chain of D flip-flops (Part 12) wired so that each stage's output feeds the next stage's input — every clock edge, data shifts one position down the chain, exactly like the 2-stage example used back in Part 4's event scheduler notes.

Shift registers are classified by how data goes in and how data comes out — Serial or Parallel, for each side. This note covers the two serial-input types: SISO and SIPO.

SISO — Serial-In, Serial-Out

Data enters one bit at a time and exits one bit at a time, delayed by N clock cycles for an N-bit register — the simplest possible shift register, often used purely as a delay line.

module siso #( parameter N = 4 )( input wire clk, rst_n, input wire sin, // serial data in output wire sout // serial data out ); reg [N-1:0] shreg; always @(posedge clk or negedge rst_n) begin if (!rst_n) shreg <= {N{1'b0}}; else shreg <= {shreg[N-2:0], sin}; // shift left, new bit enters LSB end assign sout = shreg[N-1]; // oldest bit exits from the MSB end endmodule
🔹 Key Points
  • {shreg[N-2:0], sin} is the classic shift-in idiom — concatenate all bits except the last, then tack the new bit on at the entry end.
  • Only two signals are ever exposed externally — sin and sout — none of the internal bits are directly visible, true to the "serial-out" name.
  • An N-bit SISO introduces exactly N clock cycles of latency between a bit entering and that same bit appearing at sout.
SIPO — Serial-In, Parallel-Out

Data still enters one bit at a time, but every internal bit is exposed as a parallel output bus — after N clock cycles, the entire shifted-in word is available all at once. This is exactly how many UART receivers and serial-to-parallel converters work.

module sipo #( parameter N = 4 )( input wire clk, rst_n, input wire sin, // serial data in output wire [N-1:0] pout // parallel data out ); reg [N-1:0] shreg; always @(posedge clk or negedge rst_n) begin if (!rst_n) shreg <= {N{1'b0}}; else shreg <= {shreg[N-2:0], sin}; end assign pout = shreg; // entire register exposed as one bus endmodule
🔹 Key Points
  • The internal shifting logic is identical to SISO — the only difference is what's exposed on the output port.
  • pout is valid and stable to read only after N clock edges have loaded the full word — reading it early gives a partially-shifted, meaningless value.
  • A common addition is a bit counter or a "data valid" flag that pulses once every N cycles to tell downstream logic exactly when to sample pout.
SIPO with a "Data Ready" Flag

Adding a small counter (Part 8's for-loop concept doesn't even apply here — this is a simple up-counter) lets the module signal exactly when a full word has been shifted in.

module sipo_valid #( parameter N = 4 )( input wire clk, rst_n, input wire sin, output wire [N-1:0] pout, output reg valid ); reg [N-1:0] shreg; reg [$clog2(N):0] count; always @(posedge clk or negedge rst_n) begin if (!rst_n) begin shreg <= {N{1'b0}}; count <= 0; valid <= 1'b0; end else begin shreg <= {shreg[N-2:0], sin}; if (count == N-1) begin valid <= 1'b1; count <= 0; // reset for next word end else begin valid <= 1'b0; count <= count + 1; end end end assign pout = shreg; endmodule

★ $clog2(N) computes the minimum bits needed to count up to N — a handy system function for sizing counters parametrically instead of hardcoding a width.

Important Warnings
⚠️ Reading SIPO's pout Mid-Shift Gives Garbage

Without a valid/ready signal, downstream logic has no way to know whether pout currently holds a complete, correctly-aligned word or a half-shifted mixture of old and new bits — always gate reads with a counter or external framing signal.

⚠️ Shift Direction Must Match Bit Order Expectations

The idiom {shreg[N-2:0], sin} shifts left, meaning the first bit sent in ends up at the MSB after N cycles — if your protocol expects LSB-first framing, you need {sin, shreg[N-1:1]} (shift right) instead.

⚠️ Non-Blocking Assignments Are Mandatory Here

Since every shift register in this note lives inside a posedge clk always block, all assignments must use <= per Part 3's golden rule — using blocking assignments would corrupt the shift chain by letting a later stage see an already-updated earlier stage in the same clock edge.

⚠️ SISO Delay Is Fixed by Register Width

The N clock-cycle latency of a SISO register is entirely determined by its bit width — there's no way to change the delay without changing N (parameter) or adding a variable-length structure like a FIFO, which is a very different kind of block.
