Fractional Frequency Division ➗

All About VLSI · Lecture Notes — Part 20: Dual-Modulus Averaging Method
The Idea: Average Between Two Integer Divisors

Fractional division (e.g. divide by 2.5) can't be done with a single fixed integer count. Instead, a dual-modulus approach alternates between two nearby integer divide values (e.g. divide-by-2 and divide-by-3) over a repeating pattern, so the average division ratio equals the desired fraction.

For divide-by-2.5: alternate one cycle of ÷2 and one cycle of ÷3 → average = (2+3)/2 = 2.5.

Behavioural Verilog — Divide-by-2.5 (Dual-Modulus Method)
module freq_div_2p5 ( input wire clk_in, input wire rst_n, output reg clk_out ); reg toggle_pattern; // alternates which modulus is active integer count; integer mod_val; always @(posedge clk_in or negedge rst_n) begin if (!rst_n) begin count <= 0; clk_out <= 0; toggle_pattern <= 0; end else begin mod_val = toggle_pattern ? 3 : 2; // alternate 2, 3, 2, 3... if (count == (mod_val - 1)) begin count <= 0; clk_out <= ~clk_out; toggle_pattern <= ~toggle_pattern; end else begin count <= count + 1; end end end endmodule

Over two output half-periods, the total input cycles consumed = 2 + 3 = 5, spread across 2 toggles → average full-period = 5 cycles → divide-by-2.5. Duty cycle here is not perfectly even since one half is ÷2 and the other ÷3, but the average frequency ratio is exact.

This generalizes to any integer + 0.5 divisor. Other fractions (e.g. .25, .33) need a longer repeating pattern with more than two modulus values, weighted so the long-term average matches the target ratio.

⚠️ Fractional Dividers Have Jitter

Because the instantaneous period alternates between two different integer values, a fractional divider's output has cycle-to-cycle jitter — the average frequency is exact, but individual periods are not identical. This matters for clock-sensitive applications (e.g. driving other flip-flops directly) and is usually acceptable only where average frequency, not instantaneous period, is what's needed.
