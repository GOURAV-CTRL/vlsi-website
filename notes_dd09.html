SOP & POS Forms

Digital Design — Module 02, Video 09

Once a truth table is known, a Boolean function can be written in two standard forms — Sum of Products (SOP) and Product of Sums (POS). These canonical forms are the direct bridge between a truth table and a real gate-level circuit.

1Minterms and Maxterms

For a function of n variables, each row of the truth table corresponds to one specific combination of inputs:

  • Minterm — a product (AND) term that is 1 for exactly one input combination. A variable appears un-complemented if it's 1 in that row, complemented if it's 0.
  • Maxterm — a sum (OR) term that is 0 for exactly one input combination. A variable appears un-complemented if it's 0 in that row, complemented if it's 1 (the opposite convention of minterms).

2Sum of Products (SOP)

SOP expresses a function as the OR of AND terms — literally, a "sum" of "products". It's built by OR-ing together every minterm where the output is 1.

How to build it
1. Find every row in the truth table where F = 1.
2. Write the minterm for each such row.
3. OR all those minterms together.
Example
F(A,B) = 1 for rows A=0,B=1 and A=1,B=1
F = A'B + AB

Circuit-wise, SOP maps directly onto an AND-OR two-level structure — a set of AND gates feeding a single OR gate.

3Product of Sums (POS)

POS expresses a function as the AND of OR terms — built by AND-ing together every maxterm where the output is 0.

How to build it
1. Find every row in the truth table where F = 0.
2. Write the maxterm for each such row.
3. AND all those maxterms together.
Example
F(A,B) = 0 only for row A=0,B=0
F = (A + B)

Circuit-wise, POS maps onto an OR-AND two-level structure — a set of OR gates feeding a single AND gate.

4Choosing Between Them

Rule of thumb — Use SOP when the output is 1 for fewer rows of the truth table, and POS when the output is 0 for fewer rows — whichever gives the shorter expression usually leads to a smaller circuit.
Key takeaway — SOP and POS are just two mirror-image ways of reading the same truth table — one built from the 1s, the other from the 0s — and both are the standard starting point before applying simplification techniques like K-maps.
