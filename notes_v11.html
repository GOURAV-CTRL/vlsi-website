Operators in Verilog 📒

Logical Operators

Treat the entire operand as a single boolean condition — any nonzero value is true (1), zero is false (0). Result is always 1 bit.

OperatorMeaningExample
&&Logical ANDa && b
||Logical ORa || b
!Logical NOT!a
a = 4'b0110; // nonzero → true b = 4'b0000; // zero → false a && b = 1'b0; // true AND false a || b = 1'b1; // true OR false !a = 1'b0; // NOT true !b = 1'b1; // NOT false
🔹 Where They're Used
  • Almost always inside if conditions, e.g. if (rst_n && enable).
  • Combine multiple conditions cleanly — much more readable than nested if statements.
  • Never used to combine bits of a bus — that's the job of bitwise operators (& , |), not logical ones.

★ If any bit of the operand is x and no bit is 1, the logical result itself becomes x (ambiguous truth value) — not simply 0 or 1.

Shift Operators

Move bits of a vector left or right by a specified number of positions — extremely common for building multipliers, dividers by powers of 2, and serial shift logic.

OperatorMeaningFill Bit
<<Logical left shift0
>>Logical right shift0
<<<Arithmetic left shift0
>>>Arithmetic right shiftSign bit (MSB)
reg [3:0] a = 4'b1100; a << 1 = 4'b1000; // logical left, 0 fills LSB a >> 1 = 4'b0110; // logical right, 0 fills MSB a >>> 1 = 4'b1110; // arithmetic right, sign bit fills MSB (if signed)
🔹 Logical vs Arithmetic Shift
  • Logical shifts (<<, >>) always fill new bits with 0, regardless of signed/unsigned.
  • Arithmetic right shift (>>>) fills with the sign bit — this preserves the sign of a signed number, effectively dividing it by 2 while keeping negativity.
  • Arithmetic left shift (<<<) behaves exactly the same as logical left shift — fills with 0 either way.
reg signed [3:0] s = -4'sd4; // 4'b1100 s >>> 1 = 4'b1110; // -2, sign preserved s >> 1 = 4'b0110; // +6, sign lost — wrong for signed math!

★ Shifting is much cheaper in hardware than multiply/divide — a << n is equivalent to a × 2ⁿ, and a >> n is a ÷ 2ⁿ (unsigned).

Important Warnings
⚠️ Don't Confuse & with &&

Writing if (a & b) instead of if (a && b) is a very common bug — the bitwise version can produce a nonzero (true-ish) result even when the "intended" logical condition should be false, since it depends on which specific bits overlap.

⚠️ Use Arithmetic Shift Only on Signed Vectors

The >>> operator only preserves sign correctly if the vector is declared signed; on an unsigned reg/wire, >>> behaves exactly like a plain logical >> (fills with 0).

⚠️ Shift Amount Can Be a Variable

Unlike some HDLs, Verilog allows the shift amount to be a signal, not just a constant, e.g. a >> sel — useful for barrel shifters, but synthesizes to extra mux logic, so it's not "free" in hardware.
