What Dataflow Modelling Is

Dataflow modelling describes hardware as continuous assignments between signals, using the assign keyword — instead of gates (structural) or always blocks (behavioural). It directly expresses how data flows through combinational logic as Boolean/arithmetic expressions.

Key keyword: assign — the left-hand side must be a net (wire), not a reg, and the assignment is continuously active — it re-evaluates any time a right-hand-side signal changes, just like real combinational hardware.

Basic Examples
// 2-to-1 MUX, dataflow style assign y = sel ? a : b; // Full adder, dataflow style assign sum = a ^ b ^ cin; assign cout = (a & b) | (b & cin) | (a & cin); // 4-bit ripple carry using dataflow assign {cout, sum} = a + b + cin;

Notice there's no always block, no sensitivity list, no procedural statements — just an expression that continuously drives a wire. This maps very directly to combinational gate-level hardware.

Dataflow vs Behavioural — Quick Contrast

⚠️ Dataflow Is Combinational Only

Plain assign statements can only model combinational logic — there's no clock, no memory. Sequential elements (flip-flops, registers, FSMs) require behavioural modelling with clocked always blocks.
