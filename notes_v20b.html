All About VLSI · Lecture Notes — Part 14: Shift Registers
Completing the Shift Register Family

Part 13 covered the two serial-input types — SISO and SIPO. This note covers the two parallel-input types — PISO (parallel-in, serial-out) and PIPO (parallel-in, parallel-out) — completing all four combinations.

PISO — Parallel-In, Serial-Out

An entire N-bit word is loaded in one clock cycle, then shifted out one bit per cycle afterward — this is exactly the core of a UART transmitter, which loads a byte in parallel and serializes it bit by bit onto a single TX line.

module piso #( parameter N = 8 )( input wire clk, rst_n, input wire load, // 1 = parallel load, 0 = shift input wire [N-1:0] pin, // parallel data in output wire sout // serial data out ); reg [N-1:0] shreg; always @(posedge clk or negedge rst_n) begin if (!rst_n) shreg <= {N{1'b0}}; else if (load) shreg <= pin; // parallel load, overrides shifting else shreg <= {shreg[N-2:0], 1'b0}; // shift left, fill with 0 end assign sout = shreg[N-1]; // MSB shifts out first endmodule
🔹 Key Points
  • load is a mode-select input — when high, the whole register snapshots pin in a single cycle instead of shifting.
  • The if-else if priority means load always wins over shifting — a very common and intentional pattern for control signals.
  • After loading, it takes exactly N clock cycles (with load held low) to fully shift the word out through sout.
PISO with a "Transmission Done" Flag

Just like Part 13's SIPO with a valid flag, a real PISO usually needs to signal when it has finished shifting out the whole word, so the controller knows when it's safe to load new data.

module piso_done #( parameter N = 8 )( input wire clk, rst_n, input wire load, input wire [N-1:0] pin, output wire sout, output reg done ); reg [N-1:0] shreg; reg [$clog2(N):0] count; always @(posedge clk or negedge rst_n) begin if (!rst_n) begin shreg <= {N{1'b0}}; count <= 0; done <= 1'b1; end else if (load) begin shreg <= pin; count <= 0; done <= 1'b0; end else if (!done) begin shreg <= {shreg[N-2:0], 1'b0}; if (count == N-1) done <= 1'b1; else count <= count + 1; end end assign sout = shreg[N-1]; endmodule
PIPO — Parallel-In, Parallel-Out

The simplest of the four — an entire word loads in on one clock edge, and the entire word is visible on the output immediately after, with no serial shifting at all. This is really just an N-bit register with a load enable.

module pipo #( parameter N = 8 )( input wire clk, rst_n, input wire load, input wire [N-1:0] pin, output reg [N-1:0] pout ); always @(posedge clk or negedge rst_n) begin if (!rst_n) pout <= {N{1'b0}}; else if (load) pout <= pin; // load == 0: pout holds its value (no shifting happens in PIPO) end endmodule

★ A PIPO register doesn't technically "shift" anything internally — it earns its place in the shift-register family purely because it's built from the same flip-flop array and is often reused/multiplexed as part of a universal shift register (see the next note).

Important Warnings
⚠️ load Must Have Clear Priority Over Shift

In a PISO, writing the if-else chain in the wrong order (checking the shift condition before load) can cause a requested parallel load to be silently skipped for one cycle — always give load the highest priority branch.

⚠️ Loading Mid-Shift Discards Unsent Bits

If load is asserted before a PISO has finished shifting out its previous word, whatever bits hadn't been transmitted yet are silently overwritten — always check a done/busy flag before triggering a new load in a real transmitter.

⚠️ PIPO Is Not "Doing Nothing" Just Because It Doesn't Shift

It's tempting to think a PIPO register is trivial since there's no shifting — but it still needs the same careful reset and enable/load handling as any other flip-flop bank, and is the piece every universal shift register's "parallel load mode" ultimately reduces to.

⚠️ Non-Blocking Assignments, Same Rule as Always

Every register here lives inside a posedge clk always block, so — same golden rule from Part 3 — all assignments must use <=, keeping load and shift operations race-free relative to each other.
