✅ Modules in Verilog

A module is the basic building block of every Verilog design — it's like a "box" that performs a specific function.

  • Every module has a name, a list of ports (inputs/outputs), and a body with logic inside.
  • Modules can be connected together — this is called instantiation, where one module is used inside another (hierarchy).
  • Defined using the keywords module ... endmodule.
module and_gate (input a, input b, output y); assign y = a & b; endmodule

★ Think of a module as a reusable component — just like a LEGO block you can reuse in bigger designs.

✅ Number Representations

Verilog supports multiple number bases to represent values — useful for writing testbenches and constants clearly.

  • Binary (b) — base 2, only 0s and 1s. Example: 4'b1010
  • Octal (o) — base 8, digits 0–7. Example: 3'o17
  • Decimal (d) — base 10, normal counting numbers. Example: 8'd25
  • Hexadecimal (h) — base 16, digits 0–9 and A–F. Example: 8'hFA

General format → <size>'<base><value> — the size tells the number of bits, and the base letter tells how to read the value.

★ If size is not given, Verilog assumes a default width (simulator dependent) — always better to specify it!

✅ Comments in Verilog

Comments help document code so it's easier to read and understand — they are ignored by the simulator/compiler.

  • Single-line comment → starts with // and continues till the end of the line.
  • Multi-line comment → starts with /* and ends with */, can span many lines.
// this is a single-line comment /* this is a multi-line comment */

★ Good comments explain the "why" not just the "what" — helps a lot when revisiting old RTL code!
