Frequency Division by Odd Numbers 🌙

All About VLSI · Lecture Notes — Part 18: Behavioural Modelling, Counter Method
Why Odd Division Is Different

For even N, N/2 is a whole number, so a single posedge counter toggling every N/2 cycles gives a clean 50% duty cycle. For odd N, N/2 is not a whole number — a single-edge counter can still divide the frequency, but the output duty cycle will not be 50% (it comes out uneven, e.g. one half-cycle longer than the other).

This simplest odd-division method uses only posedge and accepts the non-50% duty cycle — useful when duty cycle doesn't matter, only the divided frequency does.

Behavioural Verilog — Simple Odd Divider (Duty Cycle Not 50%)
module freq_div_odd_simple #( parameter N = 5 // Odd divide factor (3, 5, 7, ...) )( input wire clk_in, input wire rst_n, output reg clk_out ); integer count; always @(posedge clk_in or negedge rst_n) begin if (!rst_n) begin count <= 0; clk_out <= 0; end else begin if (count == (N - 1)) begin count <= 0; end else begin count <= count + 1; end if (count == 0) clk_out <= 1; else if (count == (N/2)) // integer division truncates clk_out <= 0; end end endmodule

Here clk_out stays high for ⌈N/2⌉ cycles and low for ⌊N/2⌋ cycles — close to 50%, but not exact, because N doesn't split evenly.

⚠️ Duty Cycle Skew Grows Less Significant as N Increases

For small odd N (like 3), the duty cycle skew is very noticeable (33% vs 67%). For larger odd N, the skew becomes a smaller fraction of the period — but it's still not exactly 50%. If exact 50% is required (common for clock distribution), see Part 19's posedge+negedge technique.
