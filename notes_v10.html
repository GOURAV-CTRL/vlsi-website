Operators in Verilog 📒

All About VLSI · Lecture Notes — Part 1: Arithmetic & Bitwise
Arithmetic Operators

Operate on the numeric value of operands, treating them as whole numbers — result width follows the widest operand (with some rounding rules for division).

OperatorMeaningExample
+Additiona + b
-Subtractiona - b
*Multiplicationa * b
/Divisiona / b
%Modulus (remainder)a % b
**Power (exponent)a ** b
🔹 Behaviour Notes
  • + and - map directly to adder/subtractor hardware — cheap to synthesize.
  • * (multiplication) is expensive in hardware — synthesizes to a multiplier, uses more area/timing than add/sub.
  • / and % (division/modulus) are the most expensive and are often non-synthesizable unless the divisor is a power of 2 — mostly used in testbenches.
  • ** (power) works only in simulation for most tools — rarely synthesizable.
wire [3:0] a = 4'd6, b = 4'd3; wire [4:0] sum = a + b; // 9 wire [4:0] diff = a - b; // 3 wire [7:0] prod = a * b; // 18 wire [3:0] quo = a / b; // 2 wire [3:0] rem = a % b; // 0

★ If either operand is x or z in any bit, the entire arithmetic result becomes unknown (x) — this trips up a lot of beginners in simulation.

Bitwise Operators

Operate bit-by-bit on the operands — each output bit depends only on the corresponding input bits at that same position, no carry propagation.

OperatorMeaningExample
&Bitwise ANDa & b
|Bitwise ORa | b
^Bitwise XORa ^ b
~^ / ^~Bitwise XNORa ~^ b
~Bitwise NOT~a
a = 4'b1100; b = 4'b1010; a & b = 4'b1000; a | b = 4'b1110; a ^ b = 4'b0110; ~a = 4'b0011;
🔹 Key Point — Width Matters
  • If operand widths don't match, Verilog zero-extends the smaller one before doing the bit-by-bit operation.
  • Bitwise operators map directly to gate-level hardware (AND gates, OR gates, etc.) — very cheap to synthesize, unlike arithmetic * or /.
  • Result width = width of the largest operand (no extra carry bit, unlike + which can grow by 1 bit).

★ A very common student bug: using bitwise & / | when a logical && / || was actually intended (or vice versa) — they behave very differently on multi-bit vectors.

Bitwise vs Logical — Quick Contrast
AspectBitwise (&, |, ^)Logical (&&, ||)
Operates onEach bitWhole value as true/false
Result widthSame as operand widthAlways 1 bit
Example (4'b0110 op 4'b0000)0110 & 0000 = 00000110 && 0000 = 0 (false)

This distinction becomes very important once you start writing if conditions and reduction checks on multi-bit signals.

Important Warnings
⚠️ Avoid /, %, and * for Synthesis Where Possible

Division and modulus by a non-power-of-2 value often can't be synthesized, or ends up creating huge combinational logic. Prefer shifts (>>, <<) for power-of-2 division when writing RTL.

⚠️ Signed vs Unsigned Arithmetic

By default, operands are treated as unsigned unless declared signed. Mixing signed and unsigned operands in one expression silently forces everything to unsigned, which can quietly break negative-number logic.

⚠️ Watch Out for x Propagation

Both arithmetic and bitwise operators propagate x/z — a single unknown bit in either operand can turn the entire result unknown, especially in arithmetic operations. Always check for uninitialized signals when debugging unexpected x in waveforms.
