Gate Level Modeling & Vivado 📒

All About VLSI · Verilog + Tool Walkthrough Notes
An introduction to Gate Level Modeling in Verilog, plus a practical walkthrough of the Xilinx Vivado design tool interface — from writing primitive-based code to viewing the synthesized schematic.
✅ Primitives

Gate level modeling uses Verilog's built-in library components called primitives to build digital designs directly out of logic gates — no custom logic equations needed.

and or not nand nor xor xnor buf

Verilog also provides tri-state primitives, used to model buffers that can be enabled/disabled:

  • buffif1 → passes input to output when control = 1
  • buffif0 → passes input to output when control = 0
  • notif1 → passes inverted input when control = 1
  • notif0 → passes inverted input when control = 0
✅ Syntax Rules

General primitive instantiation syntax:

primitive_name instance_name (output, input1, input2, ...);

Example — a basic 2-input AND gate:

and a1 (y, a, b); // y = a & b

For tri-state logic, the order changes slightly — it follows output, input, control:

bufif1 b1 (out, in, ctrl); // out = in only when ctrl = 1, else high-impedance 'z'

★ The instance name is optional for some simulators, but it's good practice to always name your gate instances for readability and debugging.

✅ Design Examples
🔹 Half Adder

Built using just one xor gate (for Sum) and one and gate (for Carry).

module half_adder (input a, b, output sum, cout); xor x1 (sum, a, b); and a1 (cout, a, b); endmodule
🔹 Full Adder

A more complex design — combines multiple gates with internal wire declarations to connect intermediate signals between gates, improving readability.

module full_adder (input a, b, cin, output sum, cout); wire w1, w2, w3; xor x1 (w1, a, b); xor x2 (sum, w1, cin); and a1 (w2, a, b); and a2 (w3, w1, cin); or o1 (cout, w2, w3); endmodule

★ Notice how internal wires (w1, w2, w3) carry signals between gates that aren't directly exposed as ports — this is key to structuring readable gate-level code.

✅ Vivado Tool Interface Walkthrough
🔹 Project Setup
  • Create a new RTL Project from the Vivado welcome screen.
  • Add and select your project sources (Verilog files) during setup.
  • Configure the design environment — target part/board, default language, etc.
🔹 Workflow Management
  • Flow Navigator → the left-side panel guiding you through the full flow (synthesis → implementation → bitstream).
  • Project Manager → window where sources, constraints, and settings are managed.
  • Design Sources vs Simulation Sources → Design Sources are the actual RTL for synthesis; Simulation Sources (e.g. testbenches) are only used for simulation, not synthesized into hardware.
🔹 Synthesis and Schematic
  • Run Elaborated Design to generate and visualize the schematic view of your RTL before full synthesis.
  • If multiple modules exist, set the correct module as 'Top' of the hierarchy — this tells Vivado which module is the actual top-level design to synthesize and display.
  • Right-click module → Set as Top

✅ Extra Tips — When to Use Gate Level Modeling
  • Useful for understanding the low-level structure of digital circuits and bridging schematic-to-code thinking.
  • Rarely used for large designs in industry — RTL (behavioral/dataflow) coding with always/assign is preferred for scalability.
  • Still important for interviews and for understanding what synthesis tools generate "under the hood".
  • Pairs well with schematic viewing in Vivado — a great way to visually confirm your gate-level code matches intended logic.
