Universal Shift Register 🔄

All About VLSI · Lecture Notes — Part 15: Shift Registers
What Makes It "Universal"

A universal shift register combines every mode from Parts 13–14 — SISO, SIPO, PISO, and PIPO — into a single module, selectable at runtime via a mode-select input. Instead of building four separate modules, one flexible register handles them all.

modeOperation
00Hold (no change)
01Shift right (serial in from the right)
10Shift left (serial in from the left)
11Parallel load
4-Bit Universal Shift Register — Behavioral Code

This directly combines Part 8's case statement style with the shift idioms from Parts 13–14 — one always @(posedge clk) block, one case on mode, four possible next-states for the register.

module universal_shift_reg #( parameter N = 4 )( input wire clk, rst_n, input wire [1:0] mode, // 00=hold 01=shift-right 10=shift-left 11=load input wire sin_right, // serial in for right-shift input wire sin_left, // serial in for left-shift input wire [N-1:0] pin, // parallel data in output reg [N-1:0] q // current register value (also PIPO output) ); always @(posedge clk or negedge rst_n) begin if (!rst_n) begin q <= {N{1'b0}}; end else begin case (mode) 2'b00: q <= q; // hold 2'b01: q <= {sin_right, q[N-1:1]}; // shift right, new bit enters MSB 2'b10: q <= {q[N-2:0], sin_left}; // shift left, new bit enters LSB 2'b11: q <= pin; // parallel load endcase end end endmodule
🔹 Key Points
  • All four modes live in one always block, selected by a clean case statement — exactly the pattern from Part 8.
  • Both shift idioms from Part 13 reappear here, just now selectable rather than hardwired to a single direction.
  • q doubles as both the shifting register and the parallel output — no separate output port needed, since q is already the full internal state.
How One Module Replaces All Four Types
Desired BehaviorHow to Get It
SISOHold mode=01 (or 10) continuously; read q[0] or q[N-1] as serial out
SIPOHold mode=01/10 for N cycles, then read entire q as parallel output
PISOPulse mode=11 once to load, then switch to mode=01/10 to shift out
PIPOPulse mode=11 to load; read q immediately as parallel output

★ This is exactly why real hardware designs favor a universal shift register over four separate modules — one block, controlled purely by mode, can be reused across completely different datapath roles.

Adding a Global Enable (Common Real-World Addition)

Many practical universal shift registers add a top-level enable signal so the entire register can be frozen regardless of mode — useful for pausing a datapath without touching the mode control lines.

always @(posedge clk or negedge rst_n) begin if (!rst_n) q <= {N{1'b0}}; else if (en) begin case (mode) 2'b00: q <= q; 2'b01: q <= {sin_right, q[N-1:1]}; 2'b10: q <= {q[N-2:0], sin_left}; 2'b11: q <= pin; endcase end // en == 0: q holds automatically, no case needed end
Important Warnings
⚠️ Mode Encoding Must Be Documented Clearly

Because mode is just a 2-bit code with no inherent meaning, every user of this module needs a clear reference for which value means shift-left vs shift-right vs load — mixing up the encoding is a very common integration bug.

⚠️ case Already Covers All 4 Combinations

Since mode is exactly 2 bits, all 2² = 4 combinations are enumerated in the case statement above — no default branch is strictly required, but adding one is still good defensive practice if mode could ever be driven to x/z.

⚠️ Simultaneously Changing mode and Expecting Same-Cycle Effect

Because this is a clocked, non-blocking always block, changing mode combinationally takes effect starting from the next clock edge, not instantly — this is standard synchronous behavior, but worth remembering when writing a testbench that toggles mode.

⚠️ Still Just Flip-Flops Underneath

Despite the added flexibility, a universal shift register is still built from the same D flip-flop array from Part 12 — the "universal" part is entirely in the combinational mux logic feeding each flip-flop's D input, selected by mode.
