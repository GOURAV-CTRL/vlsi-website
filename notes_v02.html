Verilog Data Types & Vectors 📒

All About VLSI · Lecture Notes (Part 2)
✅ Net Data Types

Used mainly to represent physical interconnections between modules — they don't store a value on their own, they just carry whatever is driven onto them.

🔹 wire
  • The standard / default net type in Verilog.
  • Driven continuously — its value always follows whatever is connected to it (like a gate output or assign statement).
  • Default value = 'z' (high impedance) when undriven.
  • It is an unsigned data type.
wire a, b, y; assign y = a & b;
🔹 wor & wand
  • Special nets used when multiple drivers are connected to the same net (multi-driver nets).
  • wor → resolves conflicts by performing a logical OR of all driver values.
  • wand → resolves conflicts by performing a logical AND of all driver values.

★ Without wor/wand, multiple drivers on a plain wire usually cause a contention (unknown 'x').

✅ reg Data Type
  • Used to store data inside procedural blocks (always / initial).
  • Despite the name, reg does not always mean a flip-flop — it just means "assigned inside a procedural block."
  • It is an unsigned data type.
  • Default value = 'x' (unknown).
reg [3:0] count; always @(posedge clk) count <= count + 1;
✅ Integer & Real
  • integer → a 32-bit signed data type. Used for counting, looping, and indexing. Cannot hold fractional values.
  • real → stores floating-point / decimal numbers, 64-bit. Generally non-synthesizable — used mainly for testbenches / calculations, not real hardware.
integer i; for (i = 0; i < 8; i = i + 1) ... real delay_time = 2.5;
✅ Vectors

A vector is simply a collection of bits grouped together under one signal name.

reg [2:0] a; // 3-bit vector
  • Bit indexing → each bit of the vector can be accessed individually, e.g. a[0], a[2].
  • Big-endian → declared as [0:2], MSB is on the right (index 0 = leftmost/first bit declared).
  • Little-endian → declared as [2:0], MSB is on the left — this is the more common convention used in practice.
  • Vector slicing → lets you access a specific range of bits, e.g. a[2:1] picks bits 2 and 1 only.
DeclarationStyleMSB position
[2:0]Little-endianLeft (bit 2)
[0:2]Big-endianRight (bit 2)
⚠️ Important Warnings
⚡ Verilog is Concurrent

All statements (outside sequential procedural blocks) execute in parallel, not top-to-bottom like software. This can cause conflicts if multiple drivers/blocks try to control the same signal without careful design.

⚡ Width Mismatches

Unlike VHDL (which throws a compile error), Verilog silently truncates — it will simply slice/keep the LSBs when assigning a bigger value to a smaller width signal. This is a very common beginner bug since no error or warning is given.

⚡ Always Size + Base Your Constants

Always write constants with their size and base, e.g. 3'b011 instead of just 3. This avoids ambiguity and unexpected default-width behavior.
