2's Complement Arithmetic & BCD Addition

Digital Design — Module 01, Video 03

With signed numbers already represented in 2's complement, the next step is learning how addition and subtraction actually work on them — and how normal binary addition needs a correction step when the operands are BCD (Binary Coded Decimal) instead of pure binary.

1Addition in 2's Complement

Two 2's complement numbers are added exactly like unsigned binary numbers — no special circuitry is needed. Any carry generated out of the MSB is simply discarded.

Example: 5 + (−3)
5 = 0101, −3 = 1101 (2's complement of 3 in 4 bits)
0101 + 1101 = 10010 → discard the carry-out bit → result = 0010 = 2 ✓

2Subtraction via 2's Complement

Subtraction A − B is performed as A + (2's complement of B). This is exactly why 2's complement is so valuable — a subtractor never needs to be built separately from an adder.

Steps
1. Take the 2's complement of B (invert all bits, then add 1).
2. Add the result to A using ordinary binary addition.
3. Discard any carry out of the MSB.

3Overflow Detection

Overflow can only happen when both operands share the same sign but the result comes out with the opposite sign — this is the simplest rule of thumb to check by hand.

Rule — Overflow occurs if the carry into the sign bit is different from the carry out of the sign bit.

4BCD Addition

BCD encodes each decimal digit (0–9) using its own 4-bit binary pattern. Since a 4-bit nibble can represent 0–15 but BCD only uses 0–9, adding two BCD digits with plain binary addition can produce an invalid result that needs correcting.

Correction Rule
Add the two 4-bit BCD digits using normal binary addition.
If the 4-bit sum is greater than 9, or a carry-out was generated, add 0110 (6) to that nibble.
This correction pushes the result into the next decimal digit correctly and generates the proper carry into the next BCD digit position.
Example: 8 + 7 in BCD
1000 + 0111 = 1111 (15 in binary, invalid as a single BCD digit)
1111 + 0110 = 10101 → carry = 1 (tens digit), remaining nibble = 0101 (5)
Result reads as 15 — correct decimal sum ✓
Key takeaway — Ordinary binary addition works directly for 2's complement numbers, but BCD addition needs a +6 correction step whenever a digit's sum exceeds 9 or produces a carry, since BCD only ever uses 10 of the 16 possible 4-bit patterns.
