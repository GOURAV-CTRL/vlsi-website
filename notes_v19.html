Flip-Flop vs Latch — Quick Recap

Where Part 11's SR latch was level-sensitive (transparent whenever enabled), every flip-flop here is edge-sensitive — it only samples its input at the exact instant of a clock edge (posedge clk), and holds that value rock-solid the rest of the cycle.

This is exactly why flip-flops, not latches, are the standard storage element in synchronous RTL design — they're immune to the glitch-through-while-enabled problem latches have.

D Flip-Flop — Behavior & Code

The simplest and most common flip-flop — output q simply copies input d on every rising clock edge. No other logic involved.

clkdq (next)
↑ (rising edge)00
↑ (rising edge)11
no edgexholds previous q
module d_ff ( input wire clk, rst_n, d, output reg q ); always @(posedge clk or negedge rst_n) begin if (!rst_n) q <= 1'b0; // async active-low reset else q <= d; // sample d on every rising edge end endmodule

★ Note the non-blocking (<=) assignment — required per the golden rule from Part 3, since this is a clocked/sequential always block.

T Flip-Flop — Behavior & Code

The toggle flip-flop — when T = 1, q flips to its opposite value on every clock edge; when T = 0, q holds. This is the natural building block for counters, since each bit toggles at half the frequency of the one before it.

Tq (next)
0q (no change)
1~q (toggle)
module t_ff ( input wire clk, rst_n, t, output reg q ); always @(posedge clk or negedge rst_n) begin if (!rst_n) q <= 1'b0; else if (t) q <= ~q; // toggle // t == 0: no assignment -> q keeps its value (still a proper FF, not a latch, // because this is inside a clocked always block, not combinational) end endmodule

★ A T flip-flop is really just a D flip-flop with d = q ⊕ t wired in front of it — many textbooks build T-FFs exactly that way instead of writing new sequential logic.

JK Flip-Flop — Behavior & Code

The most flexible classic flip-flop — it behaves like an SR latch's edge-triggered cousin, but fixes the forbidden-state problem: when both inputs are 1, instead of being invalid, it toggles.

JKq (next)Meaning
00qHold
010Reset
101Set
11~qToggle — no longer forbidden!
module jk_ff ( input wire clk, rst_n, j, k, output reg q ); always @(posedge clk or negedge rst_n) begin if (!rst_n) q <= 1'b0; else begin case ({j, k}) 2'b00: q <= q; // hold 2'b01: q <= 1'b0; // reset 2'b10: q <= 1'b1; // set 2'b11: q <= ~q; // toggle endcase end end endmodule

★ Unlike the SR latch's S=R=1 hazard from Part 11, the JK flip-flop's J=K=1 case is perfectly well-defined and safe — this is precisely the historical motivation for inventing the JK flip-flop.

This is a classic digital design exercise — since all flip-flops ultimately store one bit, any type can be converted into any other purely by feeding the right combinational logic into its inputs.

Important Warnings
⚠️ Async Reset Belongs in the Sensitivity List

An asynchronous reset like negedge rst_n must appear in the always block's event list alongside posedge clk — leaving it out means reset would only take effect on the next clock edge, defeating the purpose of "asynchronous."

⚠️ Always Use Non-Blocking Here

Every flip-flop above lives inside a posedge clk always block, so per Part 3's golden rule, all assignments must use <= — using = here would create the classic race-condition risk between chained flip-flops covered back in Part 4's event scheduler notes.

⚠️ A T Flip-Flop with T Tied High Is Just a Toggle Counter Bit

Tying t = 1 permanently turns the T-FF into a simple divide-by-2 circuit — chaining several such bits together, each driven by the previous stage's q, is exactly how a basic ripple counter is built.

⚠️ JK's Toggle Case Still Needs a Stable Clock Edge

Even though J=K=1 is well-defined (unlike the SR latch hazard), a JK flip-flop still only toggles once per clock edge — holding J=K=1 across many clock cycles produces a clean toggle sequence, not runaway oscillation, because sampling is edge-triggered, not level-triggered.
